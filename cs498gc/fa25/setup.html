ROS2 Setup

Development Environment Configuration for CS498GC

Complete setup guide for Ubuntu 22.04, ROS2 Humble, and mobile robotics development tools required for course assignments.

üñ•Ô∏è Step 1: Ubuntu 22.04 LTS Installation

Option A: Native Installation (Recommended)

Install Ubuntu 22.04 LTS as your primary OS or dual-boot system.

  • Download Ubuntu 22.04 LTS from ubuntu.com
  • Create a bootable USB drive using tools like Rufus or Etcher
  • Install Ubuntu with at least 50GB of free disk space
  • Ensure your system has at least 8GB RAM for optimal performance
Ubuntu Installation Tutorial
Option B: Virtual Machine

Run Ubuntu 22.04 in a virtual machine using VirtualBox, VMware, or Parallels.

  • Allocate at least 4GB RAM and 50GB storage to the VM
  • Enable hardware acceleration if available
  • Install VirtualBox Guest Additions for better performance
Note: VM performance may be slower, especially for Gazebo simulation.

ü§ñ Step 2: ROS2 Humble Installation

Official ROS2 Humble Installation

Follow these commands to install ROS2 Humble on Ubuntu 22.04:

# Ensure system is up to date
sudo apt update && sudo apt upgrade -y

# Install required tools
sudo apt install curl gnupg lsb-release -y

# Add ROS2 GPG key
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key | sudo apt-key add -

# Add ROS2 repository
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# Update package index
sudo apt update

# Install ROS2 Humble Desktop
sudo apt install ros-humble-desktop -y

# Install additional development tools
sudo apt install python3-pip python3-colcon-common-extensions -y

‚öôÔ∏è Step 3: Environment Setup

Configure ROS2 Environment

Set up your shell environment to automatically source ROS2:

# Add ROS2 setup to your bashrc
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc

# Source the current session
source ~/.bashrc

# Verify installation
ros2 --version

# Test with a simple example
ros2 run demo_nodes_cpp talker

üõ†Ô∏è Step 4: Mobile Robotics Packages

Install Required Packages for Course

Install additional packages needed for mobile robotics development:

# Navigation and SLAM packages
sudo apt install ros-humble-navigation2 ros-humble-nav2-bringup -y

# Gazebo simulation
sudo apt install ros-humble-gazebo-* -y

# TF2 and geometry packages
sudo apt install ros-humble-tf2-tools ros-humble-tf-transformations -y

# Robot state publisher and joint state publisher
sudo apt install ros-humble-robot-state-publisher ros-humble-joint-state-publisher -y

# Visualization tools
sudo apt install ros-humble-rviz2 ros-humble-rqt* -y

# Python dependencies
pip3 install numpy scipy matplotlib transforms3d

üéØ Step 5: Assignment 4 Specific Setup

Husky and UR3 Robot Packages

Install packages required for the mobile manipulator assignment:

# Install Husky robot packages
sudo apt install ros-humble-husky-* -y

# Install Universal Robots packages

üß™ Step 6: Testing Your Setup

Verify Everything Works

Run these commands to test your complete setup:

Test ROS2 Core
# Terminal 1
ros2 run demo_nodes_cpp talker

# Terminal 2
ros2 run demo_nodes_py listener
Test Gazebo Simulation
# Launch Gazebo
gazebo --verbose

# Test RViz2
rviz2
üí° Expected Results:
  • Talker/listener should show message exchange
  • Gazebo should open with an empty world
  • RViz2 should open with default visualization

‚ùì Troubleshooting

Make sure you've sourced the ROS2 setup: source /opt/ros/humble/setup.bash
Add it to your ~/.bashrc for automatic sourcing.

Try: export GAZEBO_MODEL_PATH=/usr/share/gazebo-11/models
For VM users: Ensure 3D acceleration is enabled.

Run: sudo apt update && rosdep update
Then reinstall the missing package: sudo apt install ros-humble-[package-name]
